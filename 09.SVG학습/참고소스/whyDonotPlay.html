직접 열거나 주소를 직접 입력하면 자동재생이 되지 않는 이유는 아래와 같다!
그래서 아래의 2.(1) 항목의 이유로 링크를 만들어서 페이지를 호출하면 자동재생이 됨!

아래 중요부분 번역:

신규 동작
알고 있을지 모르지만, 웹 브라우저는 사용자 경험을 개선하고, 애드 블로커 설치를 최소화하도록 유도하고,
비싸거나 제한된 네트워크에서 데이터 소비를 줄이기 위해 엄격한 자동 재생 정책을 추진하고 있습니다.
이러한 변경은 사용자에게 재생을 보다 효과적으로 제어하도록 하고, 제공자들이 합법적으로 사용하는 사례로
이익을 주기 위함입니다.

크롬의 자동 재생 정책은 매우 단순합니다:

1. 음소거된 자동 재생은 언제나 허용됩니다.
2. 소리가 있는 자동재생은 다음의 경우 허용됩니다:
(1) 유저가 도메인과 인터랙션 했을 때(클릭, 탭 등)
(2) 데스크탑에서, 사용자의 Media Engagement Index의 임계 값(threshold)이 넘었을 때.
사용자가 이전에 소리와 함께 비디오를 재생했었다는 것을 의미합니다.

(3) 사용자가 모바일에서 자신의 홈 화면에 사이트를 추가했거나,
데스크탑에서 PWA가 설치한 경우 상단 프레임은 하위의 iframe에 소리가 있는
자동 재생 권한을 위임할 수 있습니다.
예)
<iframe src="https://cross-origin.com/myvideo.html" allow="autoplay">

2019년 3월 19일 예정된 파이어폭스 버전66 릴리즈부터
웹페이지에서 자동 재생되는 사운드나 사운드 포함된 자동재생 동영상은 차단됩니다.
음성이 나오지 않는 동영상은 예외적으로 허용됩니다.

데스크탑 버전과 모바일 버전 파이어폭스 브라우저에 동시에 적용됩니다.
자동 재생이 차단된 사운드나 동영상은 사용자가 재생할 수 있고, 사용자 설정으로 이 옵션을 끌 수도 있습니다.

HTML 문서 안의 모든 요소에 적용되기 때문에 회피할 수 있는 방법은 없습니다.
사용자가 url창 맨 앞 버튼을 클릭하여 풀어주어야합니다
자동재생으로 사운드가 포함된 동영상 광고도 모두 해당됩니다.
